# GridMonitor Telegram-бот

Бот моніторингу електромережі через хмару **FSolar**: опитує інвертор і миттєво
надсилає в Telegram повідомлення про **зникнення/появу мережі** + заряд батареї.
Працює 24/7 на сервері — без Apple-акаунта, без лімітів iOS.

## Команди
- `/start` — підписатися на сповіщення
- `/status` — поточний стан (мережа + батарея)
- `/history` — останні події
- `/stop` — відписатися

## Швидкий старт

### 1. Створити бота
У Telegram → **@BotFather** → `/newbot` → отримаєте **токен** (`123456:ABC...`).

### 2. Налаштувати `.env`
```bash
cd bot
cp .env.example .env
# відредагуйте .env: токен, логін/пароль FSolar, серійник
```
`ALLOWED_CHAT_ID` можна лишити порожнім — після першого `/start` бот напише ваш chat_id,
впишіть його в `.env` і перезапустіть (тоді бот відповідатиме лише вам).

### 3. Запустити
**Docker (рекомендовано):**
```bash
docker compose up -d --build
docker compose logs -f
```
**Або без Docker:**
```bash
pip install -r requirements.txt
set -a; . ./.env; set +a        # підвантажити змінні (Linux/macOS)
python src/bot.py
```

## Деплой (працює всюди завдяки Docker)

| Платформа | Як |
|---|---|
| **VPS** (Hetzner, DigitalOcean…) | `git clone` → `.env` → `docker compose up -d`. Найбільше контролю. |
| **Fly.io / Railway** | Підтримують Dockerfile; задайте змінні оточення в дашборді. Є безкоштовні тарифи. |
| **Raspberry Pi** | `docker compose up -d`. ⚠️ Має бути на UPS/захищеному виводі інвертора. |

> ⚠️ **Не хостьте вдома без UPS:** якщо сервер живиться від мережі, він вимкнеться разом зі
> світлом — і ви не отримаєте сповіщення про відключення. Хмарний хостинг надійніший.

## Важливо знати
- **Залежність від звʼязку:** бот бачить відключення, лише поки інвертор шле дані у FSolar.
  Ваш off-grid інвертор на батареї — донгл живиться, але **роутер теж має бути на
  захищеному виводі**, інакше дані не дійдуть до хмари під час відключення.
- **Період:** хмара FSolar оновлюється ~раз на 5 хв — це стеля свіжості; опитуємо кожні 3 хв.

## Безпека
Облікові дані й токен — лише в `.env` (не комітиться). Бот відповідає лише `ALLOWED_CHAT_ID`.

## Тести
```bash
pip install pytest && pytest bot/tests -q
```
Покривають логіку переходів мережі та мапінг знімка (grid-on/off, SOC). Запускаються в CI
(`.github/workflows/bot-ci.yml`).

## Як це влаштовано
```
src/
  config.py    — змінні оточення
  fsolar.py    — клієнт FSolar (RSA-логін, snapshot, alarms) + parse_snapshot
  detector.py  — правило переходу мережі
  storage.py   — SQLite (підписники, останній стан, історія)
  bot.py       — команди + фоновий поллер (надсилає сповіщення при зміні)
```
